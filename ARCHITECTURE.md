# 项目架构总结

## 🎯 项目目标

设计一个 Obsidian 第三方插件，利用 Excalidraw 的 JSON 绘图原理，通过 AI 自动生成符合规范的 JSON 内容来实现自动绘图。用户只需要说清楚需求，AI 负责生成图表。

## 📋 核心功能

### 1. AI API 配置系统（✅ 已完成）
- **文件**: `src/config/AIConfigManager.ts`
- **功能**:
  - 支持多个 AI 服务切换（DeepSeek、OpenAI、阿里云百炼、火山引擎）
  - 每个服务配置独立的 API Key 和参数
  - 自动生成服务特定的请求头和 payload 格式
  - 验证配置的有效性

### 2. 图表生成引擎（✅ 已完成）
- **文件**: `src/generator/ExcalidrawGenerator.ts`
- **功能**:
  - 将用户描述转换为 AI prompt
  - 调用 AI API 生成 Excalidraw JSON
  - 验证和规范化 JSON 数据
  - 支持坐标归一化和元素清理
  - 维护生成历史用于重新生成

### 3. 样式管理系统（✅ 已完成）
- **文件**: `src/styles/StyleManager.ts`
- **功能**:
  - 定义 5 种预设绘图风格：手绘、交叉阴影、细线条、暗黑、极简
  - 对话框中可选择风格
  - 将样式应用到所有元素
  - 支持自定义样式扩展

### 4. 数据验证工具（✅ 已完成）
- **文件**: `src/utils/ExcalidrawValidator.ts`
- **功能**:
  - 验证 Excalidraw 元素的有效性
  - 清理和规范化元素数据
  - 计算元素边界框
  - 支持缩放和移动操作
  - 提取和生成代码块

### 5. 用户界面（✅ 已完成）
- **文件**: `src/ui/DiagramInputModal.ts`
- **功能**:
  - 输入框：描述图表需求
  - 下拉框：选择绘图风格
  - 生成按钮：触发 AI 生成
  - 重新生成按钮：保持风格，重新调用 AI
  - 修改描述按钮：编辑需求后重新生成
  - 插入到笔记按钮：将图表嵌入当前笔记

### 6. 主插件入口（✅ 已完成）
- **文件**: `main.ts`
- **功能**:
  - 加载和保存插件设置
  - 注册命令和功能区图标
  - 设置选项卡配置 API 密钥
  - 打开生成对话框

## 🔄 工作流程

```
用户打开插件
    ↓
选择或输入需求 + 选择风格
    ↓
点击"生成"按钮
    ↓
AIConfigManager 验证 API 配置
    ↓
ExcalidrawGenerator 构建 prompt
    ↓
调用选定的 AI API
    ↓
AI 返回 JSON
    ↓
ExcalidrawValidator 验证和清理
    ↓
StyleManager 应用风格
    ↓
显示预览和操作选项
    ↓
用户可以：
├── 插入到笔记（转换为代码块）
├── 重新生成（同样的描述，新的图表）
└── 修改描述（调整需求重试）
```

| 提供商 | 配置项 | 说明 |
|--------|--------|------|

| OpenAI | API Key | GPT 系列模型 |
| 阿里云百炼 | API Key + App ID | 阿里云通义千问 |
| 火山引擎 | API Key | 抖音字节跳动 |

## 🎨 支持的绘图风格

1. **手绘风格** - 随意自然的外观（roughness: 2）
2. **交叉阴影** - 带有阴影的正式风格（fillStyle: cross-hatch）
3. **细线条** - 干净简洁的设计（roughness: 0）
4. **暗黑主题** - 深色背景专业风格
5. **极简风格** - 最小化设计

## 🔐 安全考虑

- API 密钥在设置中作为密码输入（type="password"）
- 密钥存储在本地 Obsidian 数据中
- 不上传或记录用户内容
- 仅与配置的 AI 服务通信
